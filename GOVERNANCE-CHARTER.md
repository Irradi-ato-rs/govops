# Governance Charter — Irradi.ato.rs GovOps

This charter defines the **mandate, scope, and limits** of the Irradi.ato.rs GovOps authority.

---

## 1. Mandate

GovOps exists to:

- Define governance intent
- Interpret governance rules
- Maintain governance clarity over time
- Provide escalation and explanation
- Support audit and external review

GovOps does **not** execute governance mechanically.

---

## 2. Scope of Authority

GovOps has authority to:

- Clarify governance semantics
- Define governance processes
- Publish adoption and usage guidance
- Classify governance questions
- Recommend governance evolution paths

GovOps does **not** have authority to:

- Bypass enforcement
- Override validation
- Merge code by authority
- Grant exceptions
- Weaken Phase‑2 invariants

---

## 3. Enforcement Boundary

Enforcement is **out of scope** for GovOps.

Enforcement is performed exclusively by:

- `rexce/validate`
- GitHub branch protection
- Xcectua orchestration

GovOps decisions **cannot** override enforcement outcomes.

---

## 4. Governance Evolution

All governance evolution must occur via:

- Pull requests
- Governance Change Proposals (GCP)
- Before/after diffs
- Evidence bundles
- Canonical validation by rexce

GovOps may advise on evolution, but **cannot apply it directly**.

---

## 5. Escalation Role

GovOps acts as the **final interpretive escalation point** for:

- Governance ambiguity
- Adoption questions
- Audit clarification
- Process disputes

Escalation does **not** imply exception‑granting.

---

## 6. Transparency

GovOps operates transparently via:

- Public documentation
- GitHub Discussions
- Published rationale

Private channels are used only for security matters.

---

## 7. Limits

GovOps authority ends where enforcement begins.

If governance behavior differs from intent:
- Enforcement wins
- Governance must evolve formally

---

**End of Charter.**
